Accordions

Accordions are elements that expand or collapse when clicked on. They allow you to hide content that you do not want to consume page space unless the user decides that it is relevant to see.

Accordion Types

There are two accordion behaviors provided.
To only allow one section to stay open at a time, specify data-accordion-type="accordion" on the <ul> element with the class="c-na-accordion".
To allow multiple information panels to stay open, specify data-accordion-type="expandable" on the <ul> element with the class="c-na-accordion" attribute. See below for a demo of each type

Alerts

There are 4 types of alerts: basic, warning, serious (failure of something, something that should not happen, or something serious like a tornado warning for a weather site), and success. No matter which variations you choose from below (borders, no border, with icons, etc.), the type of alert displayed is changed with one simple class on the alert container.

  • minimal: use the class c-na-alert--minimal
  • basic: use the class c-na-alert--basic
  • warning: use the class c-na-alert--warning
  • something serious / failure: use the class c-na-alert--serious
  • success: use the class c-na-alert--success

Note: By default, alerts have a max-width set so they will never take up the full page. To have them take up the full width of their container, add the class u-na-fw to the alert container.

Accessibility: The alerts are a11y friendly. Out of the box, all of the alerts use colors which meet the color contrast requirements to be considered accessible. They all have role="alert", and the content will be read by a screenreader even if the content within the alert (the text or links, for example) changes dynamically. The dismiss button uses a native button element, which means it can be dismissed by keyboard events when the button is in focus using the ENTER key or SPACEBAR without having to add any more JavaScript to do so. The dismiss button has a title attribute so that when sighted users hover over it, they can be sure what it does. It also has an aria-label of "dismiss alert" for screenreaders that do not read title attributes.. Finally, the dismiss button and links within the alert have a very clear focus state, which is the same as the hover state. (They should vary in the future as they can be both focused and hovered at the same time.)

For the basic and icon alert sections, you will see the default state for each type of alert. Following that will just be two examples showing variations, but every alert type can be customized in the same way.

Ripple - an effect

A Nascent UI Framework <button> with effect clipping




A Nascent UI Framework <button> without effect clipping

Requirements and Notes

  • The ripple effect can be set on most any element that can receive a click event, as long as it is non-statically positioned. That requirement is one of two reasons that there is another version of this effect ( ripple-body-child ) which does not have that requirement.
  • In order for the ripple effect to be set on an element, the element must be given a class="ripple-element".
  • You must also include the ripple.js, and ripple.css files in your application for the effect to be applied.
  • NOTE: with this version of ripple, if and element receiving the ripple effect has an accessibility outline on focus, that outline will expand with the scaling of the effect. For this reason, there is another version of this effect ( ripple-body-child ) which does not have this side-effect.
    Other than that, this method works as expected.

Defaults

  • The default color for the ripple effect is white with transparency - rgba(255, 255, 255, 0.5).
    You can customize the color with any valid hsla, or rgba color - see "Customizations" below.
  • The default effect will:
    - be circular;
    - emanate from where the mouse was clicked, and extend beyond the boundaries of the element to which the effect was applied;
    - terminate in a size twice that of the largest dimension of the width and height of the element to which it is applied;
    If you're comfortable with CSS properties, you can easily modify the scale property to produce different effect sizes and elliptical shapes by changing the values and number of parameters for the scale property in ripple.js.

Customizations

  1. Color: you can specify a custom hsla, or rgba color by including the custom data attribute data-ripple-color="hsla(h, s%, l%, a)/rgba(r, g, b, a)" to the element on which the effect is to be applied.
  2. Effect Clipping: you can specify that the effect be clipped, i.e., not extend past the boundaries of the element to which the effect will be applied, by simply adding the custom data attribute data-ripple-clip="true" to the element on which the effect is to be applied.

Scrollspy

Scrollspy, along with a "sticky" navigation (the navigation on the left-hand side of the page), is operating on this demo page.
Typical of scrollspy behavior, is that of some visually apparent state change to the navigation item that corresponds to a "spied-on" section that is in view.
Atypical of scrollspy behavior, is the fact that this version of scrollspy has the "active" navigation item remain in view while scrolling the "spied-on" sections. That is, rather than have the "active" navigation element be above or below the viewport, as you scroll lengthy "spied-on" content, the "active" navigation item will be placed in the viewport if it otherwise wouldn't be.
This is still a bit of a work-in-progress to contend with some functionality side-effects associated with the aforementioned feature.
